## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between badge text and icon |
| sm | 8px | Compact element spacing, input internal padding |
| md | 16px | Default element spacing, card padding, form field gaps |
| lg | 24px | Section padding within cards, pricing tier card internal padding |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Landing page section breaks (hero to pricing, pricing to FAQ) |
| 3xl | 64px | Page-level top/bottom padding |

Exceptions: Touch target minimum 44px height for all interactive elements (pay buttons, tier selection cards, admin approve/reject buttons) per WCAG 2.5.5.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 98%)` / `#fafafa` (gray-50) | Page background, surfaces |
| Secondary (30%) | `hsl(0 0% 100%)` / `#ffffff` (white) | Cards, pricing tiers, admin table, payment form container |
| Accent (10%) | `hsl(173 58% 39%)` / teal-600 (`#0d9488`) | Primary CTA buttons, pricing tier selection highlight border, step number badges, active nav indicator, links |
| Destructive | `hsl(0 84% 60%)` / red-500 (`#ef4444`) | Reject button (admin), error messages, payment rejection badge |

**Accent reserved for (explicit list):**
1. "Start Consultation" / "Pay Now" primary CTA button background
2. Pricing tier card selected-state border (2px solid)
3. Step number circles on landing page (1, 2, 3 flow)
4. Text links (hover underline)
5. Admin "Approve" button background

**NOT accent:** Navigation background (uses white), card backgrounds (uses white), body text (uses gray-800).

**Additional semantic colors:**
| Role | Value | Usage |
|------|-------|-------|
| Success | `hsl(142 71% 45%)` / green-500 (`#22c55e`) | Approved payment badge, session active indicator |
| Warning | `hsl(38 92% 50%)` / amber-500 (`#f59e0b`) | Pending payment badge, session expiring indicator |
| Muted text | `hsl(220 9% 46%)` / gray-500 (`#6b7280`) | Secondary descriptions, timestamps, disclaimers |

## Page Inventory

### Page 1: Landing Page (`/`)

**Layout:** Single-column, vertically scrolling, max-width 1024px centered.

**Sections (top to bottom):**
1. **Header bar** -- "TaxSpecialista Consult" wordmark left-aligned, no navigation links (single-page scroll or anchor links only for MVP)
2. **Hero** -- Display headline + subheadline + primary CTA button ("Start Consultation" scrolls to pricing)
3. **How It Works** -- 3-step horizontal cards (mobile: vertical stack): (1) Choose your plan, (2) Pay via GCash or bank transfer, (3) Get your consultation link
4. **Pricing Tiers** -- Two cards side-by-side (mobile: stacked). Each card: tier name, price, description, feature list, "Choose Plan" button. Selected state: teal-600 border
5. **Trust Signals** -- CPA credentials block + advisory disclaimer
6. **FAQ** -- Accordion (shadcn Accordion component). 5-7 questions covering: what you get, how it works, is it legal advice, how long is the session, what payment methods
7. **Footer** -- "TaxSpecialista Consult" + disclaimer + contact email + copyright

**Key interactions:**
- Tier selection highlights the card (teal border) and enables the "Pay Now" CTA
- "Pay Now" navigates to `/pay?tier={basic|comprehensive}`

### Page 2: Payment Instructions + Submission (`/pay`)

**Layout:** Single-column, max-width 640px centered, card container.

**Sections:**
1. **Selected tier summary** -- Tier name + price badge at top
2. **Payment method tabs** (shadcn Tabs) -- GCash | Bank Transfer
3. **GCash tab content** -- QR code image (from `assets/gcash-qr.png`), GCash number, amount to send
4. **Bank tab content** -- Bank name, account number, account name, amount to send
5. **Submission form** -- Reference number input (required), screenshot upload (optional, UploadThing dropzone), email input (required), submit button

**Key interactions:**
- Tab switch between GCash and Bank Transfer
- File upload dropzone with drag-and-drop + click-to-browse
- Form validation on submit (Zod): email format, reference number 6-50 chars
- Submit button shows loading spinner during submission
- On success: redirect to `/pay-submitted`

### Page 3: Payment Submitted Confirmation (`/pay-submitted`)

**Layout:** Single-column, max-width 480px centered, vertically centered on viewport.

**Content:** Success icon (Lucide `CheckCircle` in green-500), heading, body text with email highlighted, next step instruction.

**Key interactions:** None -- static confirmation. Link back to landing page in footer.

### Page 4: Consultation Gate / Placeholder (`/consult/[sessionId]`)

**Layout:** Full-width with centered content area.

**States:**
- **Valid session:** Placeholder card: "Your consultation is ready. Chat functionality coming in Phase 2." Shows session tier, expiry countdown, and email.
- **Expired session:** Read-only banner (amber background) + session expired copy + "Start New Consultation" CTA.
- **Invalid session:** Error card with invalid session copy + CTA to landing page.

### Page 5: Admin Payment Dashboard (`/admin/payments`)

**Layout:** Full-width, max-width 1200px, with a data table.

**Sections:**
1. **Header** -- "Payment Verification" heading + filter tabs: All | Pending | Approved | Rejected
2. **Data table** (shadcn Table) -- Columns: Date, Email, Tier (badge), Method (badge), Reference #, Amount, Screenshot (link/thumbnail), Status (badge), Actions
3. **Actions column** -- Pending rows: "Approve" (teal button) + "Reject" (outline destructive button). Approved/Rejected rows: status badge only.
4. **Empty state** -- When no payments match the filter

**Key interactions:**
- Tab filtering (All/Pending/Approved/Rejected)
- Approve click: immediate action with optimistic UI update (no confirmation needed -- approval is non-destructive)
- Reject click: opens Dialog with reason select (required) + confirm/cancel
- Screenshot column: click to open in new tab or Dialog lightbox
- Table rows sorted by newest first

## Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| < 640px (mobile) | Single column. Pricing tiers stack vertically. Admin table becomes card list. QR code scales to 200px width. |
| 640-1024px (tablet) | Two-column pricing tiers. Admin table at full width. |
| > 1024px (desktop) | Max-width containers centered. Landing page sections at comfortable reading width (1024px). Admin dashboard at 1200px. |

## Accessibility Contracts

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | All text meets WCAG AA (4.5:1 for body, 3:1 for large text). Teal-600 on white passes AA for large text; use teal-700 (`#0f766e`) for small text links if needed. |
| Focus indicators | Default shadcn focus ring (2px offset, ring color). Do not remove. |
| Form errors | Inline error messages below each field, associated via `aria-describedby`. Error fields get red border. |
| Touch targets | Minimum 44x44px for all buttons and interactive elements. |
| Screen reader | Payment status badges include `aria-label`. Admin actions include `aria-label` with email context (e.g., "Approve payment from user@example.com"). |
| Reduced motion | Loading spinners use CSS animation with `prefers-reduced-motion: reduce` support. |
