<decisions>
## Implementation Decisions

### Date Input Format
- **Flexible parsing**: Tools accept multiple date formats (ISO YYYY-MM-DD, "March 15, 2026", "03/15/2026") and parse internally
- User chose this over ISO-only for resilience — the AI may pass dates in various formats from conversation context
- Use a lightweight date parsing approach (Date constructor + validation, or a small utility) — no heavy date library

### Prescription Scope
- **Basic 3yr/10yr only**: NIRC Section 203 (3-year general) and Section 222 (10-year fraud/failure-to-file)
- Amended return clock reset is OUT of scope for this task
- Waiver-extended prescription is handled separately by checkWaiverValidity — no duplication
- assessmentBasisDate = LATER of (actual filing date, statutory deadline)

### Waiver Defect Rules
- Claude's Discretion: hardcode the known defect criteria from RDAO 1-2000 and RMO 20-90
- Standard defects: missing notarization, expired on execution, unauthorized signatory, missing witness, no specified expiry date, wrong tax types, not executed before original prescription expires

</decisions>

<specifics>
## Specific Ideas

- System prompt already documents exact tool signatures and when to call each tool
- calculateDeadlines takes LOA receipt date + optional NIC/PAN dates, returns deadline table
- calculatePrescription takes filing date, tax period, fraud flag, returns expiry + days remaining
- checkWaiverValidity takes waiver details, returns list of defects with legal basis
- All results are deterministic — no LLM reasoning for date math
- Existing tool pattern: src/lib/ai/tools.ts createDocumentTools() and createComputationTools() (if exists)

</specifics>
