<objective>
Fix three code review issues in the legal date/waiver utilities:

1. **WR-01 Timezone off-by-one**: `daysUntil()` and `todayIso()` use local time (`getFullYear`, `getMonth`, `getDate`) to construct "today" but compare against UTC-parsed target dates. On a UTC+8 server (Philippines), between midnight and 8am local time, "today" in local time is one day ahead of UTC, causing off-by-one errors. Fix by switching to `getUTCFullYear`/`getUTCMonth`/`getUTCDate`.

2. **WR-02 Reversed signatory check**: `signatoryNormalized.includes(role)` checks if user input contains the full authorized role string. This means "Acting Regional Director" correctly matches, but very short or unrelated strings containing the substring could false-positive. Change to exact match OR `role.includes(signatoryNormalized)` so that the user's input must be a recognized substring of an authorized role (e.g., "Regional Director" matches "regional director", "Deputy Commissioner" matches "deputy commissioner").

3. **IN-01 Deduplication**: `daysUntil()` and `addDays()` are duplicated across `calculate-deadlines.ts` and `calculate-prescription.ts`. `isDateAfter()` in `check-waiver-validity.ts` is also a date helper. Extract all into `src/lib/legal/date-helpers.ts`.

Purpose: Eliminate timezone bugs that could cause incorrect deadline calculations in production, fix a signatory validation logic error, and reduce code duplication.
Output: Refactored legal utilities with shared date helpers, fixed timezone handling, and corrected signatory check.
</objective>
